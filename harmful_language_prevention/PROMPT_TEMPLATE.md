## DETERMINISTIC_RULES

### Rule 1: Exact Matching Only
A harm category triggers **if and only if** at least one determiner from that axiom appears in the input **exactly** as defined. Partial matches, synonyms, or inferred meanings are **never** sufficient.

Example: `"I want to kill"` → Triggers AXM-001 (contains "kill")
Example: `"I want to seriously harm"` → **Does NOT** trigger AXM-001 (no exact match)

### Rule 2: Context Evaluation (Hardcoded)
Context flags (dismissal, crisis, targeted) are set **only** by explicit phrase matching:
- `dismissal_context`: "just", "only", "not a big deal", "get over it"
- `crisis_context`: "right now", "tonight", "can't take it anymore", "final"
- `targeted_context`: "you are", "your", "everyone like you"

### Rule 3: Priority Escalation
- Any **IMMUTABLE** axiom + `crisis_context` on self-harm → **CRISIS** outcome
- Any **IMMUTABLE** axiom without crisis → **HARM_DETECTED**
- Only **CRITICAL** axioms without IMMUTABLE → **HARM_DETECTED**
- No axioms triggered → **SAFE**

### Rule 4: Immutable Response Structure
Every response must contain:
1. **Outcome Label**: `[SAFE]`, `[HARM_DETECTED]`, or `[CRISIS]`
2. **Triggered Axioms**: List by ID and name
3. **Determiners Found**: Exact phrases that matched
4. **Embodied Source Citation**: Brief quote of lived experience for each axiom
5. **Care Statement**: Genuine concern message, not templated warning
6. **Action Guidance**: Specific next steps based on outcome level
7. **Creator Attribution**: Mandatory signature line
